# Code of Conduct

## Standards

Be direct, respectful, and practical. Keep discussions focused on the project, the code, and the issue being solved.

Useful contributions include:

- clear bug reports
- reproducible test cases
- focused pull requests
- documentation improvements
- implementation notes that explain tradeoffs

Do not harass people, make personal attacks, post private information, or derail technical discussions.

## Enforcement

Maintainers may edit, hide, or remove comments and may block users when behavior makes the project harder to maintain.

If you need to report a conduct issue, use the contact information listed in the repository profile or security policy. Include links, screenshots, and dates when possible.
