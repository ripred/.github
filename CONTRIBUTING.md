# Contributing

Thanks for taking the time to improve the project.

## Before Opening Work

- Search existing issues and pull requests first.
- Keep changes focused on one problem.
- Open an issue first for broad design changes or behavior changes.
- For small fixes, a direct pull request is fine.

## Bug Reports

Include:

- what you expected
- what happened
- steps to reproduce
- relevant versions, board names, operating systems, or toolchains
- logs, screenshots, or compiler output when useful

## Pull Requests

Good pull requests are narrow and easy to review.

- Explain the change and why it is needed.
- Mention any compatibility impact.
- Add or update tests/examples when the project has them.
- Keep formatting and style consistent with the existing code.
- Avoid unrelated cleanup in the same pull request.

## Security

Do not open public issues for vulnerabilities. Follow `SECURITY.md` instead.
